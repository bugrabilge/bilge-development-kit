# Research Mode Context

> Active when analyzing technologies, comparing solutions, or investigating problems.

---

## Research Principles

1. **Breadth before depth** - Survey the landscape, then dive deep
2. **Evidence-based** - Cite sources, show data, avoid assumptions
3. **Structured output** - Tables, comparisons, pros/cons format
4. **Recency matters** - Prefer 2024-2026 sources over older ones
5. **Actionable conclusions** - End with clear recommendations

## Output Format

### Technology Comparison

```markdown
## [Topic] Comparison

| Criteria | Option A | Option B | Option C |
|----------|----------|----------|----------|
| Performance | [data] | [data] | [data] |
| DX | [rating] | [rating] | [rating] |
| Community | [size] | [size] | [size] |
| Cost | [price] | [price] | [price] |
| Learning curve | [level] | [level] | [level] |

### Recommendation
[Option X] because [evidence-based reasoning].
```

### Problem Investigation

```markdown
## Investigation: [Problem]

### Symptoms
- [Observable behavior]

### Hypotheses
1. [Most likely cause] - Evidence: [...]
2. [Alternative cause] - Evidence: [...]

### Root Cause
[Confirmed cause with proof]

### Solution
[Recommended fix with trade-offs]
```

## Research Depth Levels

| Level | When | Effort |
|-------|------|--------|
| **Quick scan** | Known domain, simple question | 5 min |
| **Standard** | New library/tool evaluation | 15 min |
| **Deep dive** | Architecture decision, migration | 30+ min |

## Anti-Patterns

- Recommending without comparing alternatives
- Using outdated information (pre-2024 for fast-moving tech)
- Presenting opinions as facts
- Ignoring trade-offs or only showing positives
